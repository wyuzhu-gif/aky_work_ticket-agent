# 本 Wiki 的用途

这是一个作业票审查知识库，服务于 ai-document-review 系统的三类核心场景。

## 场景一：合规审查

根据作业票类型（permit_type）检索对应法规条款，注入 LLM 审查 prompt。

### 检索路径（按票种）

**动火作业（hot_work）：**
1. 核心条款 → [[gb30871-2022-hot-work]]（GB 30871-2022 第5章 动火作业）
2. 必查关联 → [[气体检测要求]]（5.2.4 气体分析）、[[动火作业安全措施]]（5.3 安全措施）
3. 审批要求 → [[动火作业审批要求]]（5.4 审批流程、附录A票样）
4. 专属约束 → 气体分析时效≤30min、安全措施交叉核验、监护人专职履职

**受限空间作业（confined_space）：**
1. 核心条款 → [[gb30871-2022-confined-space]]（GB 30871-2022 第6章 受限空间作业）
2. 必查关联 → [[气体检测要求]]（6.2.3 氧含量/有毒/可燃气体）、[[受限空间安全措施]]
3. 审批要求 → [[受限空间作业审批要求]]（附录B票样）
4. 专属约束 → 进入前连续检测、氧含量19.5%-23.5%、有毒气体限值

**盲板抽堵作业（blind_plate）：**
1. 核心条款 → [[gb30871-2022-blind-plate]]（GB 30871-2022 第7章 盲板抽堵作业）
2. 必查关联 → [[管道隔离要求]]、[[盲板抽堵安全措施]]
3. 审批要求 → [[盲板抽堵审批要求]]（附录C票样）
4. 专属约束 → 盲板材质/压力等级匹配、管道能量隔离确认

### 检索策略
- 合规审查必须按"核心条款→必查关联→审批要求"三步检索，不可遗漏
- 气体检测是动火和受限空间的共同必查项，检索时自动关联
- 8个审查维度（危害辨识/风险控制/现场条件/审批签字/时效性/人员资质/交叉作业/作业关闭）各有对应条款，检索时按维度聚焦

## 场景二：文档审查

对上传的作业票文档进行结构化提取和合规检查。

### 检索路径
1. 识别文档类型（动火/受限空间/盲板）
2. 按场景一的对应路径检索法规
3. 额外检索 [[XXX作业字段映射]] 了解提取字段与法规条款的对应关系

## 场景三：智能问答（SmartQuery）

用户自然语言查询作业数据，系统生成 SQL 执行查询。

### 检索路径
1. 法规类问题 → 检索标准实体页和概念页
2. 数据类问题 → 检索 [[数据库表结构说明]] 和 [[常见查询SQL]]
3. 混合问题（"有哪些不合规的动火票"） → 同时检索法规知识 + 数据库结构

### SmartQuery 知识源
- DDL（表结构） → 存入 [[数据库表结构说明]]
- 业务文档（字段说明） → 存入各概念页
- 历史 SQL 对 → 存入 [[常见查询SQL]]
